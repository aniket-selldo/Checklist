#Bulk Activities
    #Member Lists
        1.Create lists from existing lead searches, from lead IDs, or from phone numbers (XLS format only).
        2.Assign a purpose (Sales, Presales, Export, Marketing) so lists appear in the right modules.
        3.Avoid duplicates by using one consolidated list for broad blasts.
    #Email Marketing
        1.Setup: name, description, from-name/email, select campaign and reply-to, optionally add analytics parameters.
        2.Select Member List: choose one or more lists; you can save and resume later.
        3.Design: use an approved template or HTML; include an unsubscribe link.
        4.Review & Send: test to sample recipients; send now, schedule, or save draft.
    #SMS Marketing
        1.Setup: name, description, SMS type (Promotional or Transactional), link to a campaign for reply tracking.
        2.Select Member List: ensure valid phone coverage; creation of new lists is supported.
        3.Design: pick an approved SMS template; templates and headers must match DLT approvals.
        4.Review & Send: test numbers supported; send now, schedule, or save draft.
    #Bulk Activity FAQs (highlights)
        1.Promotional SMS to DND numbers will not deliver.
        2.Ensure DLT registration, header and template approvals, and exact template text/ID.
        3.For emailers, replies land on the lead activity if reply-to tracking is configured.

#Bulk Import
    #Common prerequisites
        1.Use Excel XLS (97-2003) format; include country code for phones.
        2.Never leave mandatory columns blank; remove unknown columns instead.
        3.Mandatory attributes: phone or email, campaign, source, sub-source, sales user, stage (as applicable).
    #Method 1 (single tagging context)
        1.Use when most rows share campaign/source/sub-source/sales and are assigned evenly.
        2.Provide tagging via the upload form; the sheet can omit those IDs.
        3.Resolve parser errors (dates, phone formats) before continuing.
    #Method 2 (mixed sources/users)
        1.Use when rows span different campaigns/sources/sales/stages.
        2.Include exact IDs/names in the sheet; then map headers without reselecting in the form.
        3.Fetch correct IDs for project, campaign, users before retrying.
    #Typical errors and fixes
        1.Date formats: use DD/MM/YYYY; avoid future dates.
        2.Names/IDs: ensure exact spellings and valid IDs; do not comma-separate multiple values in one cell.

#Routing
    #Core concepts
        1.Priority defines which entity (campaign/project/source/sub-source/medium) dominates rule selection.
        2.Rules assign available users; stickiness preserves owner on re-engagement; auxiliary can climb to other rules.
        3.Call forwarding hops unanswered calls per IVR settings.
    #Use cases and guardrails
        1.Keep priorities specialized→generalized; avoid overlapping/misguiding rules; cap number of priorities.
    #Troubleshooting
        1.If leads land in default or wrong rule, verify priorities, rules, auxiliary behavior, and user availability.
